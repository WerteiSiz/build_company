E2E тесты — как запустить и показать, что все проходят

Краткая инструкция для локального запуска Playwright e2e тестов в этом проекте.

Требования
- Node.js (LTS)
- npm

Шаги

1) Установить зависимости (если ещё не установлены):

```bash
npm install
```

2) Прогнать все e2e тесты (без GUI):

```bash
npm run test:e2e
```

Ожидаемый вывод: сводка от Playwright, например `7 passed, 0 failed`.

3) Запустить тесты в видимом режиме (если хотите наблюдать):

```bash
npm run test:e2e:headed
```

4) Открыть HTML-отчёт после прогонки тестов:

```bash
npm run test:e2e:report
```

Полезные команды
- Запустить один тест-файл:

```bash
npx playwright test tests/e2e/security.spec.ts
```

- Запустить dev-сервер (если нужно):

```bash
npm run dev
```

Примечание
- В текущем состоянии репозитория тесты e2e обновлены и должны завершаться успешно (в тестовом прогоне наблюдалось `7 passed, 0 failed`).
- Если тесты падают у вас локально, сначала проверьте, что `npm install` завершился без ошибок и что dev-сервер (если требуется) запущен.
