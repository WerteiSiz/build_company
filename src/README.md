
# FixFlow — управление дефектами на строительных объектах

FixFlow — демонстрационное SPA-приложение на **React** для централизованного управления дефектами на строительных объектах. Проект предназначен для демонстрации процессов контроля качества, работы с дефектами, объектами и ролями пользователей в едином интерфейсе.

Приложение включает упрощённую авторизацию, несколько уровней ролей, аналитические представления и набор e2e-тестов для проверки ключевых пользовательских сценариев.

---

## Возможности

* **Система ролей**: инженеры, менеджеры и руководители с разными правами доступа
* **Управление дефектами**: создание, просмотр, обновление статусов и контроль исправления
* **Управление объектами**: работа со строительными объектами и отслеживание их состояния
* **Аналитика и отчёты**: визуальная статистика, графики и сводная информация по проектам
* **Демо-авторизация**: предустановленные учётные записи для локальной разработки и тестирования
* **Адаптивный интерфейс**: корректная работа на десктопе и мобильных устройствах

---

## Роли пользователей

### Инженер

* Регистрация дефектов
* Обновление статусов
* Добавление комментариев
* Просмотр назначенных задач

### Менеджер

* Все возможности инженера
* Назначение исполнителей
* Контроль сроков выполнения
* Управление объектами
* Формирование отчётов по проектам

### Руководитель / Директор

* Все возможности менеджера
* Просмотр аналитики по всем объектам
* Формирование сводных отчётов
* Общий контроль процессов

## Быстрый старт

### Требования

* Node.js (LTS)
* npm

### Установка зависимостей

```bash
npm install
```

### Запуск dev-сервера

```bash
npm run dev
```

Приложение будет доступно по адресу:
[http://localhost:3000](http://localhost:3000)

### Сборка для продакшена

```bash
npm run build
```

### Просмотр собранной версии

```bash
npm run preview
```

## Демо-учётные записи (локально)

Для демонстрации и тестирования в проекте доступны предустановленные учётные записи:

* **Инженер**: `engineer` / `demo`
* **Менеджер**: `manager` / `demo`
* **Руководитель**: `director` / `demo`

Как войти:

1. Откройте `http://localhost:3000/test-login` или нажмите «Начать работу» на главной странице.
2. Введите логин и пароль из списка выше.
3. Нажмите «Войти».

> Учётные записи предназначены только для локальной разработки и демонстрации.

## Запуск E2E-тестов (Playwright)

В проекте реализованы end-to-end тесты для проверки ключевых пользовательских сценариев.

### Установка Playwright

После установки основных зависимостей необходимо установить Playwright:

```bash
npx playwright install
```

### Запуск тестов

Тесты расположены в директории `tests/e2e`.

Прогон всех e2e-тестов:

```bash
npm run test:e2e
```

Запуск в видимом браузере:

```bash
npm run test:e2e:headed
```

Просмотр HTML-отчёта:

```bash
npm run test:e2e:report
```

Если dev-сервер работает не на стандартном адресе, укажите базовый URL:

**Bash**

```bash
PLAYWRIGHT_BASE_URL=http://localhost:5173 npm run test:e2e
```

**PowerShell**

```powershell
$env:PLAYWRIGHT_BASE_URL='http://localhost:5173'; npm run test:e2e
```
