
# FixFlow - Управление дефектами на строительных объектах

Веб-приложение для централизованного управления дефектами на строительных объектах, разработанное на React.

## Функциональные возможности

- **Система ролей**: Инженеры, менеджеры и руководители с разными правами доступа
- **Управление дефектами**: Создание, отслеживание и контроль исправления дефектов
- **Управление объектами**: Контроль строительных проектов и их прогресса
- **Аналитика и отчеты**: Детальная статистика с графиками и экспорт отчетов
- **Адаптивный дизайн**: Работает на десктопе и мобильных устройствах

## Установка

1. Убедитесь, что у вас установлен Node.js (LTS) и npm.
2. Установите зависимости:
```bash
npm install
````

3. Запустите приложение:

```bash
npm run dev
```

4. Откройте браузер по адресу [http://localhost:5173/](http://localhost:5173)

## E2E тесты (Playwright)

Ниже — краткая и копируемая инструкция для локального запуска e2e тестов в этом проекте.

Требования
- Node.js (LTS)
- npm

Установка зависимостей

```bash
npm install
```

Прогон всех e2e тестов (без GUI)

```bash
npm run test:e2e
```

Ожидаемый результат: Playwright выведет итог, например `7 passed, 0 failed`.

Прогон тестов в видимом браузере

```bash
npm run test:e2e:headed
```

Открыть HTML-отчёт

```bash
npm run test:e2e:report
```

Запустить конкретный файл тестов

```bash
npx playwright test tests/e2e/security.spec.ts
```

Если нужно запустить dev-сервер перед тестами

```bash
npm run dev
```

Примечание: после внесённых правок тестовая страница авторизации (`TestLogin`) рендерится для прогонов, и в моём прогонах наблюдался `7 passed, 0 failed`.

## Демо-учётные записи (для локального тестирования)

Для быстрого входа в приложение используйте следующие тестовые учётные записи в форме `TestLogin` (или на странице `/test-login`):

- **Инженер**: логин `engineer`, пароль `demo`
- **Менеджер**:  логин `manager`,  пароль `demo`
- **Руководитель**: логин `director`, пароль `demo`

Как войти:
- Откройте `http://localhost:5173/test-login` или нажмите `Начать работу` на главной и выберите роль.
- Введите соответствующий логин и пароль в тестовой форме и нажмите `Войти`.
- Интерфейс установит роль и откроет соответствующий дашборд.

Примечание по безопасности: эти учётные записи предназначены только для локальной разработки и тестирования; не используйте их в продакшене.
